## Общее описание проекта

В рамках контрольной работы №3 было разработано прогрессивное веб-приложение (PWA) для управления заметками, включающее следующие ключевые возможности:

- **Создание и удаление заметок** с хранением в `localStorage`
- **Заметки с напоминанием** — выбор даты и времени, отложенное уведомление
- **WebSocket-синхронизация** в реальном времени между вкладками
- **Push-уведомления** через VAPID-протокол
- **Кнопка «Отложить»** в уведомлении (snooze на 5 минут)
- **Офлайн-режим** через Service Worker и Cache API
- **Установка на устройство** как нативное приложение (manifest.json)

## Практическая работа 13–14: Основы PWA

**Цель:** Создание базового PWA-приложения с App Shell-архитектурой, Service Worker и офлайн-режимом.

### Выполненные задачи:

1. **App Shell-архитектура** — создан файл `index.html` с фиксированным header, навигацией и динамически загружаемым контентом через `fetch()`.

2. **Динамическая загрузка контента** — HTML-фрагменты (`home.html`, `about.html`) загружаются асинхронно из папки `/content/` и вставляются в DOM через `innerHTML`.

3. **Манифест PWA** — файл `manifest.json` с иконками, темой, цветом statusBar, параметрами отображения (`standalone`).

4. **Service Worker — кэширование:**
   - **Install** — пре-кэширование статических ресурсов (App Shell: HTML, CSS, JS, иконки, манифест)
   - **Activate** — удаление устаревших кэшей
   - **Fetch** — стратегии:
     - **Cache First** для статики
     - **Network First** для динамического контента (`/content/`)

5. **Заметки с localStorage** — CRUD-операции: добавление, отображение, удаление заметок.

### Результат:
- Приложение работает без интернета
- Все ресурсы доступны из кэша
- Заметки сохраняются между сессиями

## Практическая работа 15–16: Сервер и WebSocket

**Цель:** Добавление серверной части (Node.js + Express) и WebSocket-синхронизации для работы в реальном времени.

### Выполненные задачи:

1. **Сервер на Express + HTTPS:**
   - HTTP-сервер на порту 3000
   - HTTPS-сервер на порту 3001 (требуется для Service Worker)
   - SSL-сертификаты через `mkcert` (доверенные локальные сертификаты)

2. **Socket.IO — WebSocket-синхронизация:**
   - Клиент отправляет событие `newTask` при создании заметки
   - Сервер рассылает `taskAdded` всем подключённым клиентам
   - Заметки синхронизируются между вкладками в реальном времени

3. **REST API:**
   - `POST /api/notes` — создание заметки (триггерит WebSocket + Push)
   - `GET /api/vapid-public-key` — получение публичного VAPID-ключа

4. **Установка зависимостей:**
   - `express` — веб-фреймворк
   - `socket.io` — WebSocket-библиотека
   - `web-push` — отправка Push-уведомлений

### Результат:
- Несколько клиентов видят заметки друг друга в реальном времени
- Сервер обрабатывает события и рассылает обновления

## Практическая работа 17–18: Push-уведомления и напоминания

**Цель:** Реализация Push-уведомлений с отложенными напоминаниями и функцией «Отложить».

### Выполненные задачи:

1. **VAPID-ключи:**
   - Генерация пары ключей через `npx web-push generate-vapid-keys`
   - Настройка `webPush.setVapidDetails()` на сервере

2. **Push-подписка на клиенте:**
   - Запрос разрешения `Notification.requestPermission()`
   - Подписка через `registration.pushManager.subscribe()` с VAPID-ключом
   - Отправка подписки на сервер (`POST /subscribe`)
   - Сохранение и восстановление подписки при перезагрузке

3. **Отложенные напоминания:**
   - Пользователь указывает дату и время напоминания
   - Клиент отправляет `socket.emit('newReminder', { id, text, reminderTime })`
   - Сервер вычисляет `delay = reminderTime - Date.now()` и запускает `setTimeout`
   - По истечении времени сервер отправляет Push всем подписчикам

4. **Service Worker — обработка Push:**
   - Отображение уведомления с заголовком, текстом, иконкой
   - Кнопка **«Отложить на 5 минут»** (действие `snooze`)
   - Кнопка **«Открыть»** (действие `view`)

5. **Обработка клика по уведомлению:**
   - При нажатии «Отложить» — `POST /snooze` на сервер
   - Сервер: `clearTimeout` старого таймера → новый `setTimeout` на 5 минут
   - При нажатии «Открыть» — переход в приложение

6. **Хранилище напоминаний:**
   - `Map<id, { timeoutId, text, reminderTime }>` на сервере
   - Автоматическое удаление после срабатывания

### Результат:
- Push-уведомления приходят даже при закрытой вкладке
- Кнопка «Отложить» переносит напоминание на 5 минут
- Полная поддержка офлайн-режима

## Решённые проблемы

### 1. Push-подписка зависала в Chrome

**Проблема:** `pushManager.subscribe()` не возвращал результат — ни успеха, ни ошибки.

**Диагностика:** Добавлены детальные `console.log` на каждом этапе `subscribeToPush()`. Выявлено, что запрос зависает на вызове `pushManager.subscribe()`.

**Причина:** Chrome не поддерживает Push API с самоподписанными SSL-сертификатами на `localhost`. Браузер молча отменяет операцию.

**Решение:**
- Использовать `mkcert` для создания доверенных локальных сертификатов
- Добавлен таймаут 10 секунд на `subscribe()` через `Promise.race()`, чтобы функция не зависала навсегда
- Добавлено подробное логирование для отладки

### 2. Service Worker и HTTPS

**Проблема:** Service Worker требует HTTPS.

**Решение:** Сервер запущен на HTTPS (порт 3001) с сертификатами от `mkcert`, которые браузер считает доверенными.

### 3. Отладка Push-подписки

**Проблема:** Неясно, на каком этапе происходит сбой.

**Решение:** Добавлено пошаговое логирование:
- `[Push] subscribeBtn: ...` — найден ли элемент
- `[Push] Notification supported: ...` — поддерживается ли API
- `[Push] Notification.permission: ...` — текущее разрешение
- `[Push] subscribeToPush вызвана` — вход в функцию
- `[Push] Service Worker ready: ...` — готовность SW
- `[Push] existingSubscription: ...` — проверка подписки
- `[Push] Создаю новую подписку...` — вызов subscribe()
- `[Push] Ответ сервера: ...` — результат отправки

## Инструкция по запуску

```bash
# 1. Установка зависимостей
npm install

# 2. Генерация VAPID-ключей (один раз)
npx web-push generate-vapid-keys

# 3. Обновить ключи в app.js и server.js

# 4. Создание доверенных SSL-сертификатов (mkcert)
mkcert -install
mkcert localhost

# 5. Запуск сервера
node server.js

# 6. Открыть в браузере:
#    https://localhost:3001
```
