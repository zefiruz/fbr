# Заметки PWA с Socket.IO и Push-уведомлениями

PWA-приложение «Заметки» с двусторонней связью через WebSocket и системными Push-уведомлениями.

## Структура проекта

```
pr_15_16/
├── index.html          # Основной HTML (App Shell)
├── app.js              # Клиентская логика + Socket.IO + Push
├── sw.js               # Service Worker (кэш + push-обработчик)
├── style.css           # Стили
├── manifest.json       # Манифест PWA
├── server.js           # Сервер (Express + Socket.IO + web-push)
├── package.json        # Зависимости
├── localhost+2.pem     # HTTPS сертификат
├── localhost+2-key.pem # HTTPS ключ
├── content/            # HTML-фрагменты
│   ├── home.html
│   └── about.html
└── icons/              # Иконки PWA
```

## Установка и запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Генерация VAPID-ключей (опционально)

```bash
npm run generate-vapid
```

Скопируйте полученные ключи в `server.js` (переменные `VAPID_PUBLIC_KEY` и `VAPID_PRIVATE_KEY`).

### 3. Запуск сервера

```bash
npm start
```

Сервер запустится на двух портах:
- **HTTP:** `http://localhost:3000`
- **HTTPS:** `https://localhost:3001`

### 4. Открытие приложения

Откройте в браузере: **https://localhost:3001**

> ⚠️ Браузер может предупредить о самоподписанном сертификате — примите риск и продолжите.

## Функционал

### WebSocket (Socket.IO)

- При добавлении заметки клиент отправляет событие `newTask` на сервер
- Сервер рассылает событие `taskAdded` всем подключённым клиентам
- Заметки синхронизируются между всеми открытыми вкладками

### Push-уведомления

- Кнопка **«Включить уведомления»** в шапке приложения
- При нажатии запрашивается разрешение и создаётся подписка
- Подписка отправляется на сервер (`/subscribe`)
- При добавлении заметки сервер отправляет Push всем подписчикам
- Кнопка **«Отключить уведомления»** удаляет подписку (`/unsubscribe`)

### Service Worker

- **Cache First** для статических ресурсов (App Shell)
- **Network First** для динамического контента (`/content/`)
- Обработчик `push` — показывает системное уведомление с текстом заметки
- Обработчик `notificationclick` — открывает приложение при клике

## API эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/vapid-public-key` | Получить публичный VAPID ключ |
| `POST` | `/subscribe` | Сохранить Push-подписку |
| `POST` | `/unsubscribe` | Удалить Push-подписку |
| `POST` | `/api/notes` | Создать заметку (WebSocket + Push) |

## Примечания

- Подписки хранятся в памяти сервера (в продакшене используйте базу данных)
- Для работы Push-уведомлений требуется HTTPS
- VAPID ключи нужно сгенерировать один раз и сохранить
- Service Worker кэширует App Shell для офлайн-работы
