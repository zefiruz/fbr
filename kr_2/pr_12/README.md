## Отчет о выполнении практических работ №7–11

# Практическая работа №7: Основы серверной части
На данном этапе был заложен фундамент приложения. Реализован REST API для работы с пользователями и товарной номенклатурой.

Аутентификация: Созданы эндпоинты для регистрации и логина. Пароли пользователей защищены с помощью алгоритмов хеширования.

Управление товарами: Реализован стандартный CRUD-интерфейс (создание, чтение, обновление, удаление).

Структура данных: Определены основные сущности: User (id, email, имя/фамилия) и Product (id, заголовок, категория, описание, цена).

# Практическая работа №8: Внедрение защищенных маршрутов
Вторая часть работы была посвящена безопасности и идентификации пользователей.

JWT-авторизация: Реализована выдача токена при входе в систему.

Приватные маршруты: Маршруты для получения детальной информации о товаре, его редактирования и удаления закрыты от неавторизованных пользователей.

Профиль пользователя: Добавлен эндпоинт /api/auth/me для извлечения данных о текущем сеансе.

# Практическая работа №9: Механизм обновления сессий (Refresh Token)
Для улучшения пользовательского опыта и повышения безопасности внедрена система двойных токенов.

Refresh Strategy: Реализована логика обновления токенов через /api/auth/refresh.

Формат ответа: Сервер возвращает пару accessToken и refreshToken, что позволяет поддерживать сессию без частого ввода логина и пароля.

# Практическая работа №10: Интеграция с фронтендом
На этом этапе backend был соединен с клиентской частью на React.js.

Интерфейс управления: Созданы страницы для просмотра каталога, добавления новых позиций и редактирования существующих товаров.

Авторизация на клиенте: Разработаны формы входа и регистрации. Реализована логика хранения токенов и автоматической переавторизации.

# Практическая работа №11: Система ролей и администрирование (RBAC)
Финальная доработка приложения, превратившая его в полноценную корпоративную систему с разделением прав.

Ролевая модель:

Гость: Только вход и регистрация.

Пользователь: Просмотр витрины.

Продавец: Наполнение и корректировка товарной базы.

Администратор: Полный доступ к товарам + управление штатом пользователей.

Панель управления пользователями: Для роли "Администратор" реализован интерфейс для мониторинга списка пользователей, изменения их данных и блокировки учетных записей.
